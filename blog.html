Discuss the scope of var, let, and const

Var
var has function scope. Variables declared with var are hoisted to the top of their containing function, which can lead to unexpected behavior.
Let
let has block scope. Variables declared with let are hoisted, but they are not initialized until the actual declaration statement is encountered.
const
const also has block scope. Variables declared with const cannot be reassigned after declaration, but their contents (if they are objects or arrays) can be modified. Using let and const is recommended over var for better scoping and predictability in modern JavaScript.

Tell us the use cases of null and undefined

Use of null:
Explicitly represents the absence of any object value or a deliberate absence of value. Used to indicate that a variable or object property has no assigned value. Use of undefined:
Indicates a variable or object property that has been declared but hasn't been assigned a value. Often used as the initial value of function parameters or variables within a function. In summary, null is typically used to represent the intentional absence of value, while undefined is used to represent the absence of an assigned value.

What do you mean by REST API?

REST API stands for Representational State Transfer Application Programming Interface. It is a set of architectural principles for designing networked applications. RESTful APIs use HTTP requests to perform CRUD (Create, Read, Update, Delete) operations on resources,typically represented in JSON or XML format. They are known for their simplicity, scalability, and statelessness, making them widely used for building web services and connecting different software systems.
